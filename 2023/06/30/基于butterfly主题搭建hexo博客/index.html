基于butterfly主题搭建hexo博客（一）

发表于2023-06-30|更新于2023-06-30|技术教程
|字数总计:983|阅读时长:3分钟|阅读量:

一、电脑端环境安装

1、安装node.js

node.js可以在官网安装安装可以选择X64msi版，省去了还要布置环境

1、官网Node.js (nodejs.org)

如果用msi版安装，可以更改路径。

便携版必然是需要配置环境变量的

我的安装目录

E:\ALLSOFT\nodejs

a)可以使用命令行窗口运行，检查是否安装

node -v
npm -v

如上图就是正常安装了

如果你用node命令行，可以实现，但是用windows自带的cmd窗口无法实现，或者coda的命令行无法实现，即可以重启电脑，就都可以实现了

如果用的是便携版的的请将node文件夹添加到系统变量中

b)在node文件夹目录下新建以下两个文件夹

切换到win管理员cmd窗口

切换到安装目录

cd E:\ALLSOFT\nodejs

新建两个文件夹

mkdir node_global
mkdir node_cache

使用npm配置运行文件夹和缓存文件夹

npm config set prefix "E:\ALLSOFT\nodejs\node_global"
npm config set cache "E:\ALLSOFT\nodejs\node_cache"

将’’E:\ALLSOFT\nodejs\node_global’’添加到以下两个系统环境变量

c）安装hexo-cli

npm i hexo-cli -g
hexo

如果运行hexo命令会报无法找到该命令等

有两种解决方案

一是重启，是因为刚安装完nodejs，环境变量没有同步

二是命令行运行

E:\ALLSOFT\nodejs\node_global\hexo

建议重启，如果第二种方案没有运行成功的话，说明环境变量咩有配置成功。

d）电脑端博客文件夹创建

选择一个放置博客文件的目录

1、空文件夹运行
mkdir hexo-blog
cd hexo-blog
npm init -y

2、初始化hexo项目
1、hexo初始化博客
hexo init myblog 
cd myblog
npm i

2.下载主题
git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

需要安装git bash，一般都会安装，可以浏览器搜索git bash，会自动安装到c盘，

好处就是可以不用自己配置环境变量

3、配置修改

修改_config.yml中的theme

想象_config.yml就是代码版的hexo类型的设置，可以更换主题

在package.json中的scripts中添加

"start":"hexo clean && hexo g && hexo s"

最好使用vscode编辑(注意json格式)，没有代码错误高亮显示就可以运行

npm start

运行结果

打开本地网站调试发现

网络搜索解决方案

管理员cmd在myblog文件夹下运行

npm install --save hexo-renderer-jade hexo-generator-feed hexo-generator-sitemap hexo-browsersync hexo-generator-archive

再运行

npm start

访问：http://localhost:4000/

即可得到正常的网页页面

二、git环境搭建

1、安装git

略过

2、设置ssh-keygen一路enter即可

git config --global user.name "yourname"
git config --global user.email youremail@example.com
ssh-keygen -t rsa -C "youremail@example.com"


git config --global core.autocrlf false  // 禁用自动转换，这个不设置后面上传时会出现警告

3、设置公钥

打开id_rsa.pub，复制里面的内容

打开github.com

创建一个以自己用户名的前缀的仓库，必须为username.github.io

打开设置

选择ssh与GPG公钥，粘贴即可，之后添加

生成如下：

4、修改配置并推送

修改_config.yml

deploy:
- type: git
  repo: 
      git@github.com:ldl2030/ldl2030.github.io.git
  branch: master

生成静态画面

hexo g

进行git配置

npm install hexo-deployer-git --save

运行推送

hexo d

会在文件夹生成一个前缀为.deploy的文件，这就是我们要上传的文件

之后登录github更改默认分支为master

就可以看到文件夹了

5.打开网站ldl2030.github.io享用吧！
